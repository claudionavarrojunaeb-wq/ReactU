Problemas encontrados y soluciones aplicadas
- Error: "Test environment ... does not export a 'getVmContext'" (incompatibilidad entre `jest-environment-jsdom` y la versión de Jest instalada).
  - Solución: instalé `jest-environment-jsdom@29.7.0` compatible con Jest 29.

- Importación de CSS / assets en componentes durante tests provocaba errores.
  - Solución: añadí `moduleNameMapper` en `jest.config.cjs` y creé `__mocks__/styleMock.js` y `__mocks__/fileMock.js`.

- Advertencia durante render en tests: "Encountered two children with the same key, `otro`."
  - Origen: `src/mocks/userTypes.ts` tiene claves duplicadas. No impidió la ejecución de tests, pero conviene corregir las claves duplicadas para evitar comportamiento inestable.

- Error TS en el test por import React no usado (JSX automatico): eliminé la importación redundante.

- Ajuste de aserción: `console.log` fue llamado con dos argumentos (`'Submitted:'` y el objeto `data`), así que ajusté la aserción para comprobar la llamada en lugar de comparar un solo string.
